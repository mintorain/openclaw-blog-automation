# 🟠 오픈클로(openclaw)로 구글 블로거(Blogger) 자동화 — 실습 워크북

> **대상:** 코딩을 한 번도 해본 적 없는 완전 초보 교육생
> **목표:** 코드를 직접 짜지 않고, **오픈클로에게 한국어로 시켜서** 구글 블로거 글쓰기를 자동화한다
> **방식:** 회색 박스의 명령을 **그대로 복사 → 오픈클로에 붙여넣기**하며 따라오면 됩니다

> 💡 먼저 `교육-실습워크북-블로그자동화.md`(티스토리/네이버)를 끝낸 교육생이라면 더 쉽습니다. 처음이어도 OK!

---

## 🌟 구글 블로거가 특별한 이유 (꼭 읽으세요)

티스토리·네이버와 **결정적으로 다릅니다.**

| 항목 | 티스토리·네이버 | 구글 블로거 |
|------|----------------|-------------|
| 공식 자동화 통로(API) | ❌ 거의 막힘 | ✅ **열려 있음 (Blogger API)** |
| 캡차(보안문자) 골치 | 😫 자주 막힘 | 😄 **거의 없음** |
| 자동화 안정성 | 화면 바뀌면 깨짐 | **튼튼함** |
| 초기 설정 난이도 | 쉬움 | ⚠️ **구글 인증 설정이 조금 번거로움** |

> 한 줄 요약: **블로거는 "정식 출입증(API)"을 발급받는 방식**이라 일단 설정만 끝나면 캡차 없이 아주 안정적입니다.
> 대신 **처음에 출입증 발급(구글 인증) 과정**이 이 워크북의 핵심 관문입니다. 오픈클로가 손잡고 안내해 줍니다.

## 🧭 이 워크북을 쓰는 법

각 단계는 이렇게 되어 있어요:

> 📋 **입력할 명령** — 오픈클로에 그대로 붙여넣을 한국어 문장
> ✅ **이렇게 되면 성공** — 무엇이 보이면 잘된 건지
> 🛟 **막히면** — 안 될 때 뭐라고 말하면 되는지

> 💡 **마음가짐:** 안 되면 오픈클로에게 **"안 돼, 쉬운 말로 설명하고 네가 고쳐줘"** 라고 하면 됩니다. 코드를 몰라도 됩니다.

- [ ] **구글 계정** (Gmail 계정)
- [ ] **블로거 블로그 1개** (없으면 [blogger.com](https://www.blogger.com) 에서 무료로 1분 만에 생성)
- [ ] 연습용 글 1~2개 (아무 내용 OK)

## 🎬 0차시 — 큰 그림 이해하기 (10분)

블로거 자동화는 이런 순서예요:

```
[1] 구글에서 "출입증(API 사용 허가)"을 발급받는다   ← 처음 한 번만, 조금 번거로움
        │
[2] 오픈클로가 그 출입증으로 블로거에 글을 올리는 프로그램을 만든다
        │
[3] 글 파일만 바꾸면 계속 자동 발행!   ← 캡차 없음, 안정적
```

> 📋 **첫 명령 — 개념 확인**
> ```
> 나는 코딩을 전혀 몰라. 구글 블로거(Blogger)에 글을 자동으로 올리는 걸
> 너에게 시켜서 만들고 싶어. 블로거는 공식 API가 있다고 들었어.
> 전체 흐름을 코딩 모르는 사람도 알게 4줄로 쉽게 설명해줘.
> ```
> ✅ **성공:** "구글에서 인증 받고 → API로 글 올린다" 같은 쉬운 설명이 나옴.

## 🛠️ 1차시 — 작업 공간 만들기

> 📋 **명령 — 프로젝트 폴더 + 도구 준비**
> ```
> 구글 블로거 자동 포스팅 연습용 작업 폴더를 만들어줘.
> 블로거 공식 API(Blogger API v3)를 쓸 거야. 필요한 프로그램
> (파이썬, 구글 API 라이브러리 등)을 네가 알아서 설치하고,
> 다 되면 "준비 끝"이라고 알려줘. 내가 할 일이 있으면 쉽게 알려줘.
> ```
> ✅ **성공:** 설치가 끝나고 "준비 끝" 안내.
> 🛟 **막히면:** `방금 나온 에러를 쉬운 말로 설명하고 네가 고쳐줘.`

## 🔑 2차시 — 구글 "출입증(API 인증)" 발급받기 ⭐가장 중요⭐

이 차시가 블로거 자동화의 **핵심 관문**입니다. 천천히 따라오세요.
구글 클라우드 화면에서 클릭하는 작업이라, **오픈클로에게 길안내를 받으며** 진행합니다.

> 📋 **명령 1 — 구글 클라우드 설정 길안내 받기**
> ```
> 블로거 API를 쓰려면 구글 클라우드(Google Cloud Console)에서 설정이 필요하다고 들었어.
> 나는 처음이라 화면이 낯설어. 아래를 코딩 모르는 사람도 따라 하게
> "클릭 순서"로 아주 자세히, 한 단계씩 알려줘:
> 1) 새 프로젝트 만들기
> 2) Blogger API 사용 설정(활성화)
> 3) OAuth 동의 화면 만들기
> 4) "데스크톱 앱"용 OAuth 클라이언트 ID 만들고, 인증 파일(JSON) 내려받기
> 각 단계마다 어떤 버튼을 누르는지 알려주고, 내가 "다음"이라고 하면 이어서 설명해줘.
> ```
> ✅ **성공:** 오픈클로가 1번부터 차근차근 클릭 순서를 알려줌. 한 단계씩 따라하면 됨.

> 🖱️ **여러분이 직접 하는 일 (브라우저에서):**
> 1. [console.cloud.google.com](https://console.cloud.google.com) 접속 → 새 프로젝트 생성
> 2. "API 및 서비스 → 라이브러리"에서 **Blogger API v3** 검색 → **사용 설정**
> 3. "OAuth 동의 화면" 구성 (외부/본인 이메일 테스트 사용자로 추가)
> 4. "사용자 인증 정보 → 사용자 인증 정보 만들기 → **OAuth 클라이언트 ID → 데스크톱 앱**"
> 5. 만들어진 인증 파일(**JSON**)을 **다운로드**

> 📋 **명령 2 — 내려받은 인증 파일 연결하기**
> ```
> 방금 구글에서 받은 인증 파일(JSON)을 내려받았어. 이 파일을 우리 작업 폴더의
> 어디에 어떤 이름으로 두면 되는지 알려줘. 그리고 이 파일은 비밀번호처럼 중요하니까
> 실수로 외부에 공유되지 않게 막아줘(.gitignore 처리).
> ```
> ✅ **성공:** "이 위치에 `client_secret.json` 으로 두세요" 같은 안내 + 보안 처리 완료.
> ⚠️ **이 JSON 파일은 비밀번호급입니다. 절대 남에게 공유 금지!**

> 🛟 **막히면 (흔한 경우):**
> - 화면 메뉴를 못 찾겠다 → `구글 클라우드 화면 메뉴 이름이 달라 보여. 지금 단계에서 정확히 어디를 눌러야 하는지 다시 알려줘.`
> - "앱이 확인되지 않음" 경고 → `구글이 "확인되지 않은 앱"이라고 경고해. 테스트 단계에서는 어떻게 넘어가면 되는지 알려줘.` (보통 "고급 → 계속" 으로 진행)

## 🔓 3차시 — 첫 로그인(인증) 한 번 하기

발급받은 출입증으로 **딱 한 번** 구글 로그인 승인을 해주면, 그 다음부터는 자동입니다.

> 📋 **명령 — 인증 실행**
> ```
> 이제 발급받은 인증 파일로 구글 로그인 승인을 한 번 진행하는 기능을 만들고 실행해줘.
> 브라우저가 열리면 내가 직접 구글 계정으로 "허용"을 누를게.
> 승인이 끝나면 다음부터는 자동 로그인되게 그 인증 결과(토큰)를 저장해줘.
> ```
> ✅ **성공:** 브라우저가 열리고 → 여러분이 구글 계정 선택 → "허용" 클릭 → "인증 완료" 메시지 + 토큰 저장.
> 🛟 **막히면:** `인증 창에서 에러가 났어. 무슨 뜻인지 쉽게 알려주고 고쳐줘.`

> 📌 **포인트:** 이 토큰 덕분에 **다음부터는 캡차도, 로그인도 신경 쓸 필요가 없습니다.** 블로거가 편한 이유!

## 🆔 4차시 — 내 블로그 번호(ID) 찾기

블로거는 글을 올릴 때 "어느 블로그인지" 번호가 필요합니다. 오픈클로가 찾아줍니다.

> 📋 **명령 — 블로그 ID 자동 조회**
> ```
> 내 구글 계정에 연결된 블로거 블로그 목록과 각 블로그의 ID(번호)를 조회해서
> 보기 좋게 보여줘. 앞으로 글을 올릴 블로그를 내가 고를 수 있게 해줘.
> ```
> ✅ **성공:** "블로그 이름 / 주소 / ID" 목록이 표로 나옴. 거기서 쓸 블로그를 고르면 됨.

> 🧪 **직접 해보기:** 예시 글의 제목과 내용을 마음대로 바꿔보세요. 그냥 글쓰기예요!

## 🚀 6차시 — 글쓰기 + 발행 자동화 (완성!)

> 📋 **명령 — 글 올리기 자동화**
> ```
> 아까 고른 블로그에, 글 보관함의 글을 하나 가져와서 블로거에 자동으로 올려줘.
> 단, 처음에는 꼭 "초안(비공개)"으로 올려서 연습부터 하자. 실행해서 결과를 보여줘.
> ```
> ✅ **성공:** "글이 초안으로 등록됐습니다" + 블로거 관리화면에 **초안 글**이 생김.
> 🛟 **막히면:** `글 올리기에서 에러가 났어. 권한 문제인지 확인하고 쉬운 말로 알려준 뒤 고쳐줘.`

> 🎉 **여기까지 오면 블로거 자동화 완성!** 글 파일만 바꾸면 계속 올릴 수 있어요.

> 📋 **(응용) 초안 확인 후 진짜 발행**
> ```
> 초안이 잘 만들어진 걸 확인했어. 이제 실제로 공개 발행하는 것도 가능하게 해줘.
> 그리고 글 여러 개를 올릴 때는 사이에 시간 간격을 두고 하루 개수도 제한해줘.
> ```

## 🧾 부록 C — 용어 사전 (1줄 설명)

| 용어 | 쉬운 뜻 |
|------|---------|
| 오픈클로 | 한국어로 시키면 코딩을 대신 해주는 AI 비서 |

| 초안(Draft) | 아직 공개 안 한 비공개 글 (연습용으로 안성맞춤) |

## ⚖️ 부록 D — 꼭 지킬 규칙 (윤리 & 안전)

1. **연습은 초안(비공개)으로.** 공개는 확인 후 마지막에.
2. **인증 파일(JSON)·토큰은 비밀번호급.** 절대 공유·업로드 금지.
3. **본인 블로그만, 적당량만.** 스팸성 대량 발행 금지.
4. **남의 글 복제 금지.** 저작권 문제가 생깁니다.
5. 자동화는 도구일 뿐, 운영 책임은 본인에게 있습니다.

---

## 🏁 수료 체크리스트

- [ ] 블로거가 왜 캡차 걱정이 적은지(공식 API) 설명할 수 있다
- [ ] 오픈클로 안내를 받아 구글 인증(OAuth)을 끝까지 발급받았다
- [ ] 내 블로그 ID를 조회할 수 있다
- [ ] 블로거에 초안 글을 자동으로 올려봤다
- [ ] 인증 파일·토큰을 안전하게 관리하는 법을 안다

---

> 📚 **강사 참고:**
> - 블로거는 **셀레니움이 아니라 공식 API + OAuth** 방식이 정답입니다(티스토리/네이버와 다른 핵심 포인트).
> - 교육생이 가장 많이 막히는 곳은 **2차시(구글 클라우드 OAuth 설정)** 입니다. 이 부분은 강사가 화면 시연으로 함께 진행하는 것을 권장합니다.
> - 같은 시리즈: `교육-실습워크북-블로그자동화.md`(티스토리/네이버), 기술 배경은 `블로그-포스팅-자동화-매뉴얼.md`.
