openclaw로 만드는 티스토리·네이버 블로그 포스팅 자동화 매뉴얼

AI 에이전트 openclaw에게 자연어 명령을 주어, 티스토리와 네이버 블로그 포스팅을 자동화하는 전체 가이드입니다. 코드를 직접 짜지 않아도, 아래 명령 흐름대로 지시하면 openclaw가 자동화 스크립트를 생성·실행합니다.

최종 업데이트: 2026-06-13

0. 시작하기 전에 — 핵심 요약

플랫폼 공식 API 2026년 현재 권장 방식 난이도
티스토리 ❌ Open API 종료(2024년 초 순차 종료) Selenium 브라우저 자동화 + (선택) ChatGPT API로 본문 생성 중
네이버 블로그 ❌ 글쓰기용 공식 API 없음 Selenium + 쿠키 기반 로그인 유지(SmartEditor iframe 제어) 상

⚠️ 중요: 두 플랫폼 모두 "글쓰기/발행" 공식 API가 사실상 없습니다. 따라서 자동화의 핵심은 브라우저 자동화(Selenium / Playwright) 입니다. openclaw에게는 "어떤 동작을 자동화할지"를 자연어로 지시하면 됩니다.

1. openclaw란 무엇이고 어떻게 명령하나

openclaw는 터미널에서 동작하는 AI 코딩 에이전트입니다. 자연어 명령을 받으면:

  1. 자동화 스크립트(Python/Node)를 생성하고
  2. 필요한 패키지를 설치하고
  3. 스크립트를 실행해 결과를 알려줍니다.

명령(프롬프트)을 잘 주는 3원칙

  1. 목표를 한 문장으로: "티스토리에 매일 1개 글을 자동 발행하고 싶어"
  2. 입력/출력을 명시: "본문은 posts/ 폴더의 .md 파일에서 읽어와"
  3. 제약을 명시: "하루 3개까지만, 발행 간격은 2시간 이상으로"

2. 사전 준비 (공통)

openclaw에게 아래처럼 환경 세팅을 먼저 지시합니다.

명령 예시 ▶
"이 프로젝트에 Python 가상환경을 만들고 selenium, webdriver-manager,
python-dotenv 패키지를 설치해줘. 그리고 .env 파일에 블로그 계정 정보를
넣을 수 있게 .env.example 템플릿을 만들어줘."

생성될 .env.example 예시:

# 절대 깃에 커밋하지 말 것 (.gitignore에 .env 추가)
TISTORY_ID=your_id
TISTORY_PW=your_password
TISTORY_BLOG_NAME=yourblog   # yourblog.tistory.com 의 yourblog 부분

NAVER_ID=your_id
NAVER_PW=your_password

OPENAI_API_KEY=sk-...        # (선택) 본문 자동 생성용

🔐 보안 원칙

  • 계정 비밀번호·API 키는 반드시 환경변수(.env) 로 관리하고 코드에 하드코딩하지 않습니다.
  • .env는 .gitignore에 추가해 깃 저장소에 올라가지 않게 합니다.
  • 2차 인증(2FA)이 켜져 있으면 자동 로그인이 막히므로, 자동화 전용 계정 또는 앱 비밀번호 사용을 검토하세요.

3. 티스토리 자동 포스팅

3-1. 왜 API가 아니라 브라우저 자동화인가

티스토리 Open API는 2024년 초에 순차 종료되어, 외부 프로그램이 토큰으로 글을 올리던 방식은 더 이상 동작하지 않습니다. 따라서 2026년 현재는 실제 사람이 브라우저를 쓰듯 로그인 → 글쓰기 → 발행을 자동화하는 Selenium 방식이 표준입니다.

참고: 과거 API 방식(https://www.tistory.com/apis/post/write)은 historical 자료로만 의미가 있습니다.

3-2. openclaw 명령 흐름

1단계 — 골격 생성

"selenium으로 티스토리에 자동 로그인하는 스크립트를 만들어줘.
.env의 TISTORY_ID/PW를 읽어서 카카오 계정 로그인 페이지를 처리하고,
로그인 성공 여부를 콘솔에 출력해줘."

2단계 — 글쓰기 자동화

"로그인된 상태에서 https://{블로그명}.tistory.com/manage/newpost/ 로 이동해서
제목과 본문을 입력하고 발행하는 기능을 추가해줘.
티스토리 에디터는 iframe 기반이니 iframe 전환 처리도 넣어줘."

3단계 — 입력 데이터 연결

"posts/ 폴더의 마크다운 파일을 하나씩 읽어서,
첫 줄(# 제목)을 글 제목으로, 나머지를 본문으로 발행하도록 해줘.
발행한 파일은 posts/done/ 으로 옮겨줘."

3-3. 티스토리 자동화 핵심 포인트 (openclaw에게 알려줄 기술 정보)

  • 로그인: 티스토리는 카카오 계정 로그인으로 통합됨 → 카카오 로그인 폼(#loginId, #password) 입력 후 제출.
  • 에디터 iframe: 글쓰기 에디터가 <iframe> 내부에 있음 → driver.switch_to.frame(...) 후 본문 입력, 끝나면 switch_to.default_content().
  • HTML 모드 활용: 마크다운/HTML 본문을 넣을 땐 에디터를 "HTML 모드"로 전환하면 서식이 안정적.
  • 발행 옵션: 공개(발행)/비공개/보호, 카테고리, 태그를 발행 직전 모달에서 설정.
  • 임시저장 팝업: 글쓰기 진입 시 "이어쓰기" 팝업이 뜰 수 있으니 자동으로 닫는 처리 필요.

3-4. (선택) 본문 자동 생성 결합

"OpenAI API(OPENAI_API_KEY)를 사용해서, 키워드 목록 keywords.txt의 각 줄을
주제로 1500자 분량 블로그 글을 생성하고, 그걸 바로 티스토리에 발행하는
파이프라인으로 연결해줘."

⚠️ 품질 관리: 하루 3~5개 수준의 발행은 비교적 안전하지만, 단기간 수십 건 대량 발행은 검색엔진 저품질(스팸) 판정 위험이 있습니다. 발행 간격(예: 2시간 이상)을 두도록 지시하세요.

4. 네이버 블로그 자동 포스팅

네이버는 자동화 탐지(캡차)가 강력합니다. 티스토리보다 까다로우므로 쿠키 기반 로그인 유지가 핵심입니다.

4-1. 가장 큰 장벽: 캡차(CAPTCHA)

  • 네이버는 프로그램적 로그인을 막기 위해 캡차를 띄웁니다.
  • 캡차를 코드로 푸는 현실적 방법은 없습니다. → 캡차가 안 뜨게 하는 전략을 씁니다.
  • 권장 전략:
    1. 쿠키 재사용: 사람이 한 번 정상 로그인한 세션의 쿠키를 저장 → 이후 자동화는 그 쿠키로 로그인 상태 유지.
    2. 사용자 프로필 로딩: 평소 쓰던 크롬 프로필(--user-data-dir)을 그대로 띄워 이미 로그인된 상태로 시작.
    3. 봇 탐지 우회 옵션: navigator.webdriver 플래그 제거 등 자동화 흔적 최소화.

4-2. openclaw 명령 흐름

1단계 — 쿠키 확보

"크롬을 사람이 쓰는 프로필로 실행해서 네이버에 수동 로그인할 수 있게 해주고,
로그인 완료되면 쿠키를 cookies.json으로 저장하는 스크립트를 만들어줘."

2단계 — 쿠키로 자동 로그인

"cookies.json을 불러와서 네이버에 로그인된 상태로 시작하는 코드를 만들어줘.
캡차를 피하기 위해 navigator.webdriver 플래그를 숨기는 옵션도 적용해줘."

3단계 — 글쓰기(SmartEditor)

"네이버 블로그 글쓰기 페이지(https://blog.naver.com/{아이디}?Redirect=Write)로 이동해서
제목과 본문을 입력하고 발행하는 기능을 추가해줘.
SmartEditor는 iframe(mainFrame) 안에 또 에디터 iframe이 중첩돼 있으니
프레임 전환을 단계적으로 처리해줘."

4-3. 네이버 자동화 핵심 포인트

  • 중첩 iframe: 글쓰기 화면은 #mainFrame 안에 SmartEditor가 들어있는 이중 iframe 구조. 프레임 전환을 정확히 해야 본문 입력이 됩니다.
  • 작성 중 팝업: "작성 중인 글이 있습니다" 복구 팝업 → 자동으로 닫기.
  • 본문 입력 방식: SmartEditor는 일반 input이 아니라 contenteditable 영역 → 클릭 후 send_keys 또는 JS로 입력.
  • 발행 모달: 발행 버튼 클릭 → 카테고리·공개설정·태그 입력하는 2차 모달 처리.
  • 속도 조절: 사람처럼 보이도록 입력 사이에 랜덤 딜레이(예: 0.5~2초) 삽입 → 탐지 회피에 유리.

4-4. 주의사항

  • 네이버 약관상 자동화는 제재(계정 정지) 대상이 될 수 있습니다. 본인 계정·소량 운영 원칙을 지키세요.
  • 2FA(2단계 인증)가 켜져 있으면 쿠키 방식이 사실상 유일한 현실적 해법입니다.
  • 캡차가 뜨면 자동화는 중단하고 사람이 1회 풀어준 뒤 쿠키를 갱신하는 흐름이 안정적입니다.

5. 스케줄링(정기 자동 발행)

명령 예시 ▶
"이 발행 스크립트를 매일 오전 9시에 자동 실행되게 설정하고 싶어.
macOS면 launchd 또는 cron, 크로스플랫폼이면 python schedule 라이브러리로
해줘. 실패하면 로그를 logs/ 폴더에 남기게 해줘."
  • 간단/상시 실행: Python schedule 라이브러리로 스크립트 내부 스케줄링.
  • OS 레벨: macOS cron/launchd, Windows 작업 스케줄러.
  • 로그 필수: 성공/실패, 발행 URL을 로그로 남겨 추적.

7. 자주 막히는 지점 & openclaw 디버깅 명령

증상 원인 openclaw에게 줄 명령
본문 입력이 안 됨 iframe 전환 누락 "글쓰기 에디터가 iframe 안에 있어. 프레임 전환 로직을 점검하고 본문이 입력되는지 스크린샷을 찍어 확인해줘."
로그인 직후 캡차 자동화 탐지 "캡차가 떴어. 쿠키 기반 로그인 방식으로 바꾸고 webdriver 흔적을 숨겨줘."
요소를 못 찾음 셀렉터 변경 "NoSuchElement 에러가 나. 현재 페이지 HTML을 덤프해서 올바른 셀렉터를 다시 찾아줘."
가끔 실패(타이밍) 로딩 대기 부족 "암묵적 대기 대신 WebDriverWait 명시적 대기로 바꿔줘."

8. 법적·윤리적 유의사항

  • 각 플랫폼의 이용약관과 robots 정책을 확인하세요. 과도한 자동화는 계정 제재 사유입니다.
  • 타인 콘텐츠 복제·대량 스팸성 발행은 저작권/저품질 문제를 일으킵니다.
  • 자동화는 반복 작업 효율화 목적에 한정하고, 운영은 본인 책임입니다.
