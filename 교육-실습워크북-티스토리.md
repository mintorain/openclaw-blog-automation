# 🟦 오픈클로(openclaw)로 티스토리 블로그 자동화 — 실습 워크북

> **대상:** 코딩을 한 번도 해본 적 없는 완전 초보 교육생
> **목표:** 코드를 직접 짜지 않고, **오픈클로에게 한국어로 시켜서** 티스토리 블로그 글쓰기를 자동화한다
> **방식:** 아래 회색 박스의 명령을 **그대로 복사 → 오픈클로에 붙여넣기**하며 따라오면 됩니다

> 🔗 **시리즈:** 이 워크북은 **티스토리** 편입니다. 같은 시리즈로 **네이버 편**, **구글 블로거 편**이 있습니다.

---

## 🧭 이 워크북을 쓰는 법 (꼭 읽으세요)

이 책은 "직접 코딩하는 법"을 가르치지 않습니다.
**"오픈클로에게 일을 시키는 법"** 을 가르칩니다. 여러분은 사장님이고, 오픈클로가 개발자입니다.

각 단계는 이런 모양으로 되어 있어요:

> 📋 **입력할 명령** — 오픈클로 입력창에 그대로 붙여넣을 한국어 문장
> ✅ **이렇게 되면 성공** — 화면에 무엇이 보이면 잘된 건지
> 🛟 **막히면** — 안 될 때 어떻게 말하면 되는지

> 💡 **가장 중요한 마음가짐:** 안 되면 그냥 오픈클로에게 **"안 돼, 이렇게 고쳐줘"** 라고 말로 하면 됩니다. 코드를 몰라도 됩니다.

---

## 📦 준비물 체크리스트

시작 전에 아래가 준비됐는지 확인하세요. (없으면 강사님께 요청)

- [ ] 컴퓨터에 **오픈클로(openclaw)** 가 설치되어 실행됨
- [ ] **티스토리 블로그 계정** (카카오 계정으로 로그인되는 것)
- [ ] (선택) 네이버 블로그 계정
- [ ] 블로그에 올릴 **연습용 글 1~2개** (아무 내용이나 OK)

> ⚠️ **연습은 반드시 "비공개" 글로 시작합니다.** 실수로 이상한 글이 공개되지 않게요.

## 🎬 0차시 — 자동화가 뭐고, 오픈클로가 왜 필요한가 (개념 10분)

### 자동화란?
매번 손으로 하던 일(로그인 → 글쓰기 → 발행)을 **컴퓨터가 알아서 반복**하게 만드는 것.

### 왜 직접 코딩 안 하나요?
블로그 자동화는 원래 파이썬 같은 프로그래밍이 필요합니다. 어렵죠.
그런데 **오픈클로가 그 코딩을 대신 해줍니다.** 우리는 한국어로 "이렇게 해줘"라고 시키기만 하면 됩니다.

### 알아둘 현실 (중요)
- 티스토리·네이버는 "글쓰기 공식 통로(API)"를 거의 닫아놨습니다.
- 그래서 자동화는 **사람이 브라우저 쓰는 걸 흉내내는 방식(셀레니움)** 으로 합니다.
- 오픈클로가 이 흉내내기 프로그램을 만들어 줍니다.

> 📋 **첫 명령 — 오픈클로와 인사하며 개념 확인하기**
> ```
> 나는 코딩을 전혀 모르는 초보야. 티스토리 블로그에 글을 자동으로 올리는 걸
> 너(오픈클로)에게 시켜서 만들고 싶어. 큰 그림을 3~4줄로 쉽게 설명해줘.
> ```
> ✅ **성공:** 오픈클로가 "로그인 → 글쓰기 → 발행을 자동화하는 프로그램을 만들어 줄게요" 같은 쉬운 설명을 해주면 OK.

---

## 🛠️ 1차시 — 작업 공간 만들기 (오픈클로가 다 해줍니다)

코딩 도구를 깔거나 설정하는 건 **전부 오픈클로에게 맡깁니다.**

> 📋 **명령 1 — 프로젝트 폴더와 환경 준비**
> ```
> 티스토리 자동 포스팅 연습용 작업 폴더를 만들어줘.
> 자동화에 필요한 프로그램(파이썬, 셀레니움 등)도 네가 알아서 설치하고,
> 설치가 다 되면 "준비 끝"이라고 알려줘. 중간에 내가 할 일이 있으면 쉽게 알려줘.
> ```
> ✅ **성공:** 오픈클로가 설치 과정을 보여주고 마지막에 "준비 끝" 비슷한 말을 함.
> 🛟 **막히면:**
> ```
> 방금 빨간 글씨(에러)가 나왔어. 무슨 뜻인지 쉽게 설명하고 네가 고쳐줘.
> ```

> 📋 **명령 2 — 계정 정보를 안전하게 넣을 곳 만들기**
> ```
> 내 블로그 아이디와 비밀번호를 코드에 직접 쓰지 말고, 안전한 설정 파일(.env)에
> 따로 넣고 싶어. 그 파일 양식을 만들어주고, 어디에 뭘 적으면 되는지 알려줘.
> 그리고 이 파일이 실수로 외부에 공유되지 않게 막아줘.
> ```
> ✅ **성공:** "여기에 아이디/비밀번호를 적으세요"라는 안내가 나옴.
> ⚠️ **보안 규칙:** 비밀번호는 **이 설정 파일에만** 적고, 채팅창에 직접 치지 마세요.

## ✍️ 2차시 — 올릴 글 준비하기

자동화는 "어떤 글을 올릴지"가 있어야 동작합니다.

> 📋 **명령 — 글 보관함 만들기**
> ```
> 내가 올릴 블로그 글을 보관할 폴더를 만들어줘.
> 글은 메모장처럼 쉽게 쓰고 싶어. 첫 줄에 제목, 그 아래에 본문을 쓰면
> 그대로 블로그에 올라가게 규칙을 정해줘. 예시 글 1개도 만들어줘.
> ```
> ✅ **성공:** `posts` 같은 폴더와 예시 글 파일이 생기고, "첫 줄=제목" 규칙을 알려줌.

> 🧪 **직접 해보기:** 예시 글을 열어 제목과 내용을 여러분 마음대로 바꿔보세요.
> 코딩이 아니라 그냥 글쓰기입니다!

---

## 🔑 3차시 — 자동 로그인 만들기

> 📋 **명령 — 로그인 자동화**
> ```
> 이제 티스토리에 자동으로 로그인하는 기능을 만들어줘.
> 아까 설정 파일에 적은 아이디/비밀번호를 사용하고,
> 로그인에 성공했는지 화면에 "로그인 성공"이라고 표시해줘.
> 나는 코딩을 모르니, 실행은 네가 직접 해서 결과를 보여줘.
> ```
> ✅ **성공:** 브라우저 창이 자동으로 열리고 로그인까지 진행됨. "로그인 성공" 메시지.
> 🛟 **막히면 (자주 발생!):**
> - **"캡차(보안문자)가 떴어요"** → 4-1로 가세요. 매우 흔한 일이고, 정상입니다.
> - **"버튼을 못 찾는대요"** →
>   ```
>   로그인 버튼을 못 찾는다는 에러가 났어. 티스토리 화면이 바뀐 것 같아.
>   지금 화면을 다시 확인해서 올바른 버튼을 찾아 고쳐줘.
>   ```

### 💡 왜 캡차가 뜰까? (꼭 알아두기)
네이버·카카오는 "기계가 로그인하는 것"을 막으려고 **보안문자(캡차)** 를 띄웁니다.
기계는 이걸 못 풉니다. 그래서 **사람이 한 번만 직접 로그인해두고, 그 상태를 재사용**하는 게 정답입니다. → 다음 차시!

---

## 🍪 4차시 — 캡차 피하기: "쿠키" 재사용 (핵심 노하우)

**쿠키 = 로그인된 상태를 기억하는 출입증.**
사람이 딱 한 번 정상 로그인해서 출입증을 받아두면, 다음부터는 비밀번호 없이 그 출입증으로 들어갑니다.

> 📋 **명령 1 — 출입증(쿠키) 저장 기능 만들기**
> ```
> 캡차 때문에 자동 로그인이 막혀. 그래서 내가 직접 한 번 로그인할 수 있게
> 브라우저를 띄워주고, 내가 로그인을 끝내면 그 로그인 상태(쿠키)를 파일로
> 저장하는 기능을 만들어줘. 그리고 그 기능을 지금 실행해줘.
> ```
> ✅ **성공:** 브라우저가 뜨면 **여러분이 직접** 평소처럼 로그인(캡차도 직접 통과) → 터미널에 Enter → "쿠키 저장 완료".

> 📋 **명령 2 — 다음부터는 출입증으로 자동 로그인**
> ```
> 이제부터는 저장한 쿠키(출입증)로 자동 로그인하게 만들어줘.
> 혹시 쿠키가 만료돼서 안 되면, "쿠키를 다시 저장하세요"라고 친절히 알려줘.
> ```
> ✅ **성공:** 비밀번호 입력 없이 로그인된 상태로 바로 진행됨.
> 🛟 **막히면:**
> ```
> "쿠키가 만료됐다"고 나와. 출입증 다시 저장하는 걸 실행해줘.
> ```

> 📌 **포인트:** 출입증은 시간이 지나면 만료됩니다. 그때는 4차시 명령 1을 다시 한 번만 하면 끝!

## 🚀 5차시 — 글쓰기 + 발행 자동화 (드디어 완성!)

> 📋 **명령 — 글 올리기 자동화**
> ```
> 로그인된 상태에서, 아까 만든 글 보관함(posts 폴더)의 글을 하나 가져와서
> 티스토리에 제목과 본문을 자동으로 입력하고 발행까지 해줘.
> 단, 처음에는 꼭 "비공개"로 올려서 연습부터 하자. 실행해서 결과 보여줘.
> ```
> ✅ **성공:** 글쓰기 화면이 자동으로 채워지고 발행됨 → 블로그 관리페이지에 **비공개 글**이 생김.
> 🛟 **막히면:**
> - **"본문이 입력이 안 돼요"** →
>   ```
>   제목은 들어가는데 본문이 안 들어가. 티스토리 글쓰기 칸은 특별한 영역(iframe)
>   안에 있다고 들었어. 그 부분을 다시 확인해서 본문이 입력되게 고쳐줘.
>   ```

> 🎉 **여기까지 오면 자동화 완성입니다!** 글 파일만 바꾸면 계속 올릴 수 있어요.

> 📋 **(응용) 여러 글을 안전한 간격으로**
> ```
> 글 여러 개를 한꺼번에 올리되, 스팸으로 보이지 않게 글 사이에 시간 간격을 두고
> 하루 3개까지만 올리도록 해줘.
> ```

## ⏰ 6차시 — 매일 자동으로 돌리기 (선택 심화)

> 📋 **명령 — 정기 실행 예약**
> ```
> 이 자동 발행을 내가 매번 누르지 않아도 매일 아침 9시에 저절로 실행되게 하고 싶어.
> 내 컴퓨터(맥/윈도우)에 맞는 방법으로 예약해주고, 잘 됐는지/실패했는지 기록(로그)도
> 남겨줘.
> ```
> ✅ **성공:** "매일 9시에 자동 실행되도록 예약했습니다" 안내 + 기록 파일 생성.

## 🌐 7차시 — 다른 블로그도 해볼까?

티스토리를 끝냈다면 다른 플랫폼도 같은 원리로 할 수 있습니다.

- **네이버 블로그**: 보안(캡차)이 더 강하지만 원리는 동일(쿠키 재사용). → **네이버 편 워크북** 참고
- **구글 블로거**: 공식 API가 있어 캡차 걱정이 적고 가장 안정적. → **구글 블로거 편 워크북** 참고

> 💡 셋 다 핵심은 같습니다: **사람이 한 번 로그인 → 그 상태를 재사용 → 오픈클로가 글쓰기 자동화.**

## 🆘 부록 A — 막혔을 때 만능 주문 모음

코드를 몰라도 이 문장들만 있으면 대부분 해결됩니다. **그대로 복붙하세요.**

| 상황 | 오픈클로에게 이렇게 말하세요 |
|------|------------------------------|
| 빨간 에러가 떴다 | `방금 나온 에러를 쉬운 말로 설명하고, 네가 직접 고쳐줘.` |
| 버튼/칸을 못 찾는다 | `화면이 바뀐 것 같아. 지금 페이지를 다시 확인해서 올바른 위치를 찾아 고쳐줘.` |
| 본문이 안 들어간다 | `글쓰기 칸이 특별한 영역(iframe) 안에 있어. 그 부분 처리를 점검해줘.` |
| 캡차(보안문자)가 떴다 | `캡차가 떴어. 쿠키(출입증) 방식으로 바꿔서 내가 한 번만 직접 로그인하게 해줘.` |
| 가끔 됐다 안 됐다 한다 | `로딩을 충분히 기다리도록 고쳐줘(명시적 대기).` |
| 뭐가 뭔지 모르겠다 | `지금 상황을 코딩 모르는 사람도 알게 1~2줄로 정리해줘.` |
| 처음부터 다시 | `지금까지 만든 걸 정리해서 어디까지 됐는지 알려주고, 다음에 뭘 하면 되는지 알려줘.` |

> 💡 **꿀팁:** "쉬운 말로", "내가 코딩 몰라", "네가 직접 해줘" 를 붙이면 오픈클로가 훨씬 친절해집니다.

## 🧾 부록 B — 용어 사전 (1줄 설명)

| 용어 | 쉬운 뜻 |
|------|---------|
| 오픈클로(openclaw) | 한국어로 시키면 코딩을 대신 해주는 AI 비서 |
| 자동화 | 손으로 하던 반복 작업을 컴퓨터가 알아서 하게 만드는 것 |
| 셀레니움(Selenium) | 사람 대신 브라우저를 클릭·입력해주는 도구 |
| 쿠키(Cookie) | 로그인 상태를 기억하는 출입증 |
| 캡차(CAPTCHA) | "사람 맞아?" 확인하는 보안문자(기계는 못 풂) |
| API | 프로그램끼리 직접 통하는 공식 통로 (티스토리·네이버는 거의 닫힘) |
| iframe | 화면 속에 들어있는 또 다른 작은 화면 (글쓰기 칸이 여기 들어있음) |
| .env 파일 | 비밀번호 같은 민감 정보를 안전하게 따로 보관하는 파일 |

## ⚖️ 부록 C — 꼭 지켜야 할 규칙 (윤리 & 안전)

1. **연습은 비공개 글로.** 공개는 확인 후 마지막에.
2. **본인 계정만, 소량만.** 과도한 자동화는 계정 정지 사유입니다.
3. **남의 글 베껴서 대량 발행 금지.** 저작권·저품질(스팸) 문제가 생깁니다.
4. **비밀번호·쿠키 파일은 절대 남에게 공유하지 마세요.** (출입증을 주는 것과 같음)
5. 자동화는 **반복 작업을 줄이는 도구**일 뿐, 운영 책임은 본인에게 있습니다.
